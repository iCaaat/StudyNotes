2.Maven生命周期

三大内建生命周期

| **生命周期名**        | **说明**                                           |
| --------------------- | -------------------------------------------------- |
| `clean`               | 清理生命周期：清除以前的构建输出                   |
| `default`（或 build） | 构建生命周期：项目的构建过程（编译、打包、测试等） |
| `site`                | 生成项目文档（比如 HTML 站点）                     |

Default生命周期

| **阶段**   | **说明**                                              |
| ---------- | ----------------------------------------------------- |
| `validate` | 验证项目结构是否正确、必要信息是否完整                |
| `compile`  | 编译 Java 源码（`src/main/java`）                     |
| `test`     | 编译并运行测试代码（`src/test/java`）                 |
| `package`  | 将项目打包（如生成 jar、war 文件）                    |
| `verify`   | 运行集成测试，检查包是否有效且符合预期                |
| `install`  | 安装打好的包到本地  `.m2/repository` 中供其他项目引用 |
| `deploy`   | 将打包好的内容部署到远程 Maven 仓库（比如私服 Nexus） |
